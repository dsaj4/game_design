## 评估维度

评分建议使用 1-5 分。分数只帮助讨论，不作为自动决策。

| 维度 | 分数 | 判断问题 | 评估 |
| --- | --- | --- | --- |
| 核心动作清晰度 |  | 玩家最常做的动作是否清楚？ |  |
| 实现可能性 |  | 以小团队/早期原型是否能做出可玩版本？ |  |
| 可玩性潜力 |  | 这个动作是否可能反复玩而不迅速疲劳？ |  |
| 决策深度 |  | 玩家是否有有意义的选择，而不是只执行流程？ |  |
| 反馈强度 |  | 玩家行为是否能得到清楚、及时、有情绪的反馈？ |  |
| 差异化 |  | 相比同类产品是否有可识别的新意？ |  |
| 市场参照 |  | 是否存在相近产品证明玩家理解这类乐趣？ |  |
| 范围控制 |  | 是否能避免一开始就依赖大量内容、数值或资产？ |  |
| 扩展潜力 |  | 如果核心成立，后续是否有自然扩展空间？ |  |
| 风险可验证性 |  | 最大风险是否能通过小原型快速验证？ |  |
